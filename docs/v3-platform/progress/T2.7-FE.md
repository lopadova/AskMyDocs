### [13:00:00] Step 1: FilterState type + chat.api.ts updates
**Action:**
- Added `FilterState` interface to `chat.api.ts` mirroring `RetrievalFilters` byte-for-byte (snake_case keys, optional arrays/strings).
- Added `isFilterStateEmpty()` + `countSelectedFilters()` helpers — used by Composer to skip the `filters` payload when empty (BE legacy fallback) and by FilterBar to render the count badge.
- Updated `chatApi.sendMessage(conversationId, content, filters?)` to accept optional filters; payload omits the `filters` key when state is empty.

### [13:05:00] Step 2: BE companion change in MessageController
**Action:** MessageController::store() must accept the same `filters` payload shape as KbChatRequest (T2.7 mutex per plan §2133). Added:
- `retrievalFilterRules()` private helper — duplicates KbChatRequest's filter-rule subset. Documented as M4 consolidation candidate.
- `buildRetrievalFilters(Request, ?string)` — constructs the DTO from the validated payload. Falls back to legacy single-project DTO when no `filters` block is sent (back-compat with pre-T2.7 callers).
- `$search->search(...)` now receives the DTO. KbSearchService.search() already accepts `?RetrievalFilters $filters` (5th param) so no service change needed.

**Suite:** Full PHPUnit run — **967/2970 green** (zero regression).

### [13:10:00] Step 3: FilterChip component
**Action:** Created `FilterChip.tsx` — single removable pill. `data-testid="filter-chip-{dimension}-{value}"` + `-remove` for the × button. Click on chip body is a no-op (only the × button removes). Remove button is keyboard-reachable (R15) with `aria-label="{Dimension} filter: {value}. Remove"`.

**Tests:** 8 cases covering dimension+value rendering, label-vs-value precedence, data-attribute exposure, click vs body-click discrimination, onRemove callback, ARIA semantics, keyboard navigation (Tab + Enter triggers remove).

### [13:15:00] Step 4: FilterPickerPopover
**Action:** Created `FilterPickerPopover.tsx` — multi-tab popover with 7 dimensions: Project, Tag, Source, Type (canonical), Folder, Date, Lang. Each tab is a panel-form. Closes on Esc + click-outside. `role=dialog` + `aria-modal=false` (non-blocking) + `role=tablist`/`role=tab`/`aria-selected` on the header. Every input has a bound `<label>` (R15).

Sub-components (file-private):
- `ProjectTab`: checkbox list of project keys, "No projects available" empty state
- `TagTab`: color-coded pill buttons (`aria-pressed` for selection)
- `SimpleListTab`: checkbox list (used for Source / Canonical / Language)
- `FolderTab`: text input + Add button + glob list with × per glob
- `DateTab`: from/to date inputs + Clear button when populated

Source/Canonical/Language enum values hardcoded — duplication with BE is the trade-off for not adding a contract-fetching layer; new enum value is a 1-line FE update.

### [13:20:00] Step 5: FilterBar (stateless parent)
**Action:** Created `FilterBar.tsx` — render N chips per dimension + "+ Filter" trigger + "Clear all" when applicable. **Stateless** — `(filters, onChange) → JSX` is the only contract. Lifts state to Composer (codified L24).

**Tests:** 12 cases covering empty state, chip rendering per dimension, count badge (counts dimensions, not values), chip removal, Clear all, popover open/close, date chips with formatted from/to labels, tag-label fallback when not in availableTags, doc-id label fallback when not in docLabels.

### [13:23:00] Step 6: Wire FilterBar into Composer
**Action:** Composer now owns `useState<FilterState>({})` and renders `<FilterBar>` ABOVE the legacy ContextChip row. Threads filters into `useChatMutation` via `mutation.mutate({ conversationId, content, filters })`. New props on Composer: `availableProjects`, `availableTags`, `docLabels` (passed through to FilterBar).

`use-chat-mutation.ts` updated: `SendMessageArgs` gains optional `filters: FilterState`.
