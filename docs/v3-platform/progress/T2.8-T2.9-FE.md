### [13:40:00] Step 1: T2.8 ‚Äî `use-mention-search` hook
**Action:** Created `use-mention-search.ts` ‚Äî TanStack Query wrapper around `GET /api/kb/documents/search` (T2.6). Query key includes the trimmed query string + project_keys, so each unique (query, scope) combination is cached for 30s. `enabled: open && query.length >= minQueryLength` keeps the hook idle when irrelevant. TanStack's signal-based abort handles racing requests automatically.

Endpoint path: `/api/kb/documents/search` (NOT `/kb/documents/search`) ‚Äî this endpoint lives in `routes/api.php` so Laravel prepends `/api`; the rest of `chat.api.ts` hits routes in `routes/web.php` (`/conversations/...`) which is why the path differs.

### [13:45:00] Step 2: T2.8 ‚Äî `MentionPopover` component
**Action:** Created `MentionPopover.tsx` ‚Äî listbox below the textarea. Up/Down/Enter/Esc keyboard nav handled at the document level (so the textarea cursor stays focused). `role=listbox` + `aria-activedescendant` per R15. Selection uses `onMouseDown + e.preventDefault()` (NOT `onClick` ‚Äî would fire after blur and break the splice). Empty/loading states surface explicitly via `data-state`.

### [13:50:00] Step 3: T2.8 ‚Äî Composer wiring
**Action:** Updated `Composer.tsx`:
- New state: `mentionQuery` + `mentionAnchorRef` (cursor index of `@` for splice).
- `onChange` detects `@<query>` context: find last `@` at-or-before cursor, ensure preceding char is whitespace (or BOS) ‚Äî else it's an email fragment. Track query as chars from `@+1` to next whitespace.
- `onMentionSelect` ‚Üí splice `@<query>` out of the textarea, add `doc.id` to `filters.doc_ids`, update `docLabelMap` so the FilterBar chip shows the friendly title.
- Wrapped textarea in a relative-positioned div so the popover renders above it.
- `aria-autocomplete="list"` + `aria-expanded` + `aria-controls` on the textarea when the popover is open.

### [14:02:00] Step 5: T2.9-FE ‚Äî `FilterPresetsDropdown` component
**Action:** Created `FilterPresetsDropdown.tsx` ‚Äî sits next to the "+ Filter" trigger in FilterBar. Three actions:
- **Load preset** ‚Üí calls `onLoad(preset.filters)`, parent (Composer) replaces live state. Menu auto-closes.
- **Save current** ‚Üí opens an inline name-input form. Disabled when current state is empty (saving emptiness is meaningless). Errors surface inline (e.g. duplicate-name 422 from BE).
- **Delete** ‚Üí requires confirm step (√ó ‚Üí Delete | Cancel inline). Single-click delete is too easy to misclick.

`useQuery({ enabled: open })` ‚Äî list fetches only when dropdown is open. `useMutation` auto-invalidates `['chat-filter-presets']` on success ‚Üí list refetches.
