### [07:45:00] Step 1: Prompt template update
**Action:** `resources/views/prompts/kb_rag.blade.php` — added a "Refusal Protocol" section between the rules block and the "Response Format" block. Three instructions:

1. Bare-sentinel emission for total refusal: `__NO_GROUNDED_ANSWER__` only, no extra wording.
2. Partial-answer encouragement: when SOME parts are answerable, answer those + explicitly note the gaps. A partial answer with skip-notes is better than full refusal.
3. Reiteration of "no undocumented assumptions" (already implicit in rule 8 but emphasized here for the refusal context).

### [07:48:00] Step 2: KbChatController sentinel detection
**Action:**
- Added `private const SELF_REFUSAL_SENTINEL = '__NO_GROUNDED_ANSWER__'` to KbChatController.
- After `$ai->chat()` returns and `$latencyMs` is computed, call `$this->isSelfRefusalSentinel($aiResponse->content)` — short-circuits to `convertSentinelToRefusal()` if true.
- `isSelfRefusalSentinel()` uses `trim($content) === self::SELF_REFUSAL_SENTINEL`. Whitespace tolerated (some providers wrap responses); substring matches NOT treated as refusal — partial answers that mention the sentinel verbatim are preserved.
- `convertSentinelToRefusal()` differs from `refusalResponse()` (T3.3) on:
  - `refusal_reason='llm_self_refusal'` (vs `'no_relevant_context'`) — different observability signal.
  - chat-log row + meta carry the REAL provider/model/token counts (LLM call paid in full).
  - `meta.refused_early=false` — LLM was reached.

### [07:51:00] Step 3: MessageController mirror

Provider/model/tokens preserved in metadata for cost attribution.

### [07:55:00] Step 4: Tests
**Action:** Created `tests/Feature/Api/KbChatSentinelTest.php` with 7 cases:
