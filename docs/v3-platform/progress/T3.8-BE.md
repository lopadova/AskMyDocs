### [08:45:00] Step 2: Per-reason resolution + fallback helper
**Action:** Added `localizedRefusalMessage($reason)` to both controllers. Resolution order:
1. Try `kb.refusal.{reason}` — per-reason copy.
2. If the translator returns the raw key (Laravel's miss sentinel), degrade to `kb.no_grounded_answer`.
3. Never leak the raw dotted key to the user.

The controllers' refusal helpers (`refusalResponse` + `convertSentinelToRefusal`) now use this resolver instead of the hardcoded `__('kb.no_grounded_answer')`. Both `KbChatController` and `MessageController` get the same surface — the helper is duplicated verbatim across them (premature to extract a trait per L22 — both are likely to merge in M4).

### [08:48:00] Step 3: Tests
**Action:** Created `tests/Feature/Localization/RefusalI18nTest.php` with 6 cases:

1. **no_relevant_context returns per-reason English copy**
2. **llm_self_refusal returns per-reason English copy**
3. **no_relevant_context returns Italian copy under it locale**
