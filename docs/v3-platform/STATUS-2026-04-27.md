## LESSONS accumulated (`docs/v3-platform/LESSONS.md`)

**M1 wave (L01..L08):**
- L01 — Pluggable pipeline: register interfaces in `PipelineRegistry::bootInstance()`, validate FQCN implements expected interface
- L02 — `final readonly` + constructor-property-promotion is the DTO shape; `@param` annotations only when collection types ambiguous
- L03 — Inline pure-PHP fixture builders > committed binaries (no `.pdf`/`.docx` in repo)
- L04 — `SourceType` enum is helper-only; column stays string for back-compat
- L05 — Polymorphic `DocumentIngestor::ingest(SourceDocument)` + `KbPath::normalize()` safety net at every entry
- L06 — `MarkdownChunker::chunkLegacy()` preserved alongside new `chunk(ConvertedDocument)` for non-pluggable callers
- L07 — Tests inspect via reflection where SQLite can't run pgvector SQL
- L08 — `$chunk->order` honored on persist (caller can override default sequence)

**M2 backend wave (L09..L16):**
- L09 — `RetrievalFilters` is a 10-dimension readonly DTO; use `forLegacyProject()` to bridge single-key APIs
- L10 — Empty `whereIn` compiles to `0 = 1` not `IN ()` — test SQL by reflection-inspection, not raw-string match
- L11 — Tag pivot has no project_key FK → must include explicit `whereColumn('kt.project_key', 'knowledge_chunks.project_key')` for tenant isolation (Copilot caught my over-confident "transitively enforced" claim)
- L12 — PHP `fnmatch + FNM_PATHNAME` does NOT support `**` cross-segment — implement glob→regex translator
- L13 — SQLite default LIKE has no escape character — pair `%`/`_` escape with explicit `ESCAPE '\\'` clause via `whereRaw`
- L14 — SQLite vs PostgreSQL LIKE case-sensitivity differs — wrap both sides in `LOWER()` for portability
- L15 — Per-user resource authorization: `where('user_id', auth()->id())` is sufficient (no Spatie policy needed). Cross-user surface = 404 (`NotFoundHttpException`), NOT 403 — avoids leaking row existence
- L16 — User uses SoftDeletes — `$user->delete()` is reversible, only `forceDelete()` triggers SQL cascade. Test BOTH paths (soft-delete-keeps + hard-delete-cascades)
