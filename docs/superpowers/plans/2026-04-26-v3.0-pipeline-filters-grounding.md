# AskMyDocs v3.0 Implementation Plan — Pipeline + Filters + Grounding

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Protocol non-negoziabili (vedi sezione "Working Conventions"):**
> 1. Ogni sub-task ha un **Verification Gate** rigoroso. Se NON passa, l'orchestratore re-dispatcha l'agente sullo stesso sub-task con il log della failure attaccato, fino a green.
> 2. Ogni sub-task **appende a `docs/v3-platform/LESSONS.md`** (bug, regola scoperta, gotcha, decisione runtime). Ogni agente successivo riceve LESSONS.md INIETTATO nel proprio prompt iniziale.
> 3. Ogni sub-task ha un **README delta** esplicito (può essere vuoto, ma il campo va valutato).
> 4. La parallelizzazione segue il **Dependency DAG** documentato. Sub-task con dipendenze risolte possono partire in parallelo.

**Goal:** Ship AskMyDocs v3.0 — la prima release della v3 enterprise platform — che porta: (a) pipeline d'ingest pluggable per markdown/text/PDF/DOCX, (b) filtri ricchi con tag/@mention/folder/doc-type nel chat composer, (c) anti-hallucination strato 0+1+2 (refusal deterministico + confidence composite). Senza connectors esterni, senza vision-LLM, senza vector-store interface — quelli arrivano in v3.1–v3.3.

**Architecture:** Tre macro-aree (Pipeline Foundation, Enterprise Filters, Grounding Tier-1) sviluppate in parallelo dove possibile, mergeate prima sul sotto-branch `feature/v3.0` e poi sul branch d'integrazione `feature/v3-platform`. Ogni macro-task = 1 PR. Quarto task macro = consolidamento (README + CLAUDE.md + skills + lessons digest).

**Tech Stack:**
- PHP 8.3, Laravel 13, PostgreSQL+pgvector, Sanctum 4
- New PHP deps: `smalot/pdfparser ^2.10`, `phpoffice/phpword ^1.3`, `symfony/process ^7.0` (per pdftotext fallback)
- React 18, TanStack Router/Query, Tailwind 3.4, Vitest + Playwright
- Test stack invariato (PHPUnit 12 + Testbench 11, Vitest, Playwright)

## 0. Working Conventions for Agents (READ FIRST)

### 0.1 Test Gate Loop Protocol

Ogni sub-task termina con un blocco `## Verification Gate`. L'agente esegue **tutti** i comandi listati. Se **anche uno solo** fallisce o l'output non corrisponde a "Expected", l'agente NON committa, NON marca il task completed, e ritorna all'orchestratore con:
- Comando che ha fallito
- Stdout + stderr
- Hypothesis di causa
- Lista delle 1-3 prossime azioni di debug

L'orchestratore **NON accetta** "ho provato ma non riesco". Re-dispatcha sullo stesso sub-task fornendo:
- Il log di failure
- Eventuale lesson appena scoperta da un altro agente sullo stesso problema (vedi LESSONS.md)
- Vincolo di non saltare il test gate

Solo dopo il green completo l'agente:
1. Aggiorna `LESSONS.md` (anche se vuoto, riga `none`)
2. Aggiorna `README.md` con il delta indicato (anche se vuoto, riga `none in this task`)
3. Committa
4. Marca il sub-task `[x]`

### 0.2 LESSONS.md Protocol

**File:** `docs/v3-platform/LESSONS.md`

**Format:**
```markdown
## [YYYY-MM-DD HH:MM] Sub-task <ID> — <Title>

**Type:** bug | rule | discovery | gotcha | decision-runtime
**Severity:** low | medium | high | critical
**Applies to:** <area> (es. "ConverterInterface implementations", "FE composer", "all PDF processing")

**Finding:**
<2-4 righe: cosa è stato scoperto>

**Why it matters:**
<1-2 righe: impatto su agenti futuri>

**How to apply:**
<1-2 righe: cosa fare diversamente>

**References:** <file:line(s) or PR/commit>
---
```

**Rules:**
- Append-only. Mai modificare entry esistenti.
- Ogni sub-task DEVE aggiungere almeno un'entry (anche `Type: discovery`, `Severity: low`, `Finding: nessuna scoperta rilevante in questo task`).
- Orchestratore in fase di dispatch DEVE includere il contenuto di `LESSONS.md` nel system prompt dell'agente come allegato `<lessons-from-previous-tasks>...</lessons-from-previous-tasks>`.
- Sub-task 4.x (consolidamento) digerisce LESSONS.md in regole permanenti su `CLAUDE.md` (R23, R24, ...) e in nuove skill su `.claude/skills/`.

### 0.3 README Delta Protocol

Ogni sub-task ha un blocco `## README Delta` che indica:
- **Section to update:** path esatto della sezione README (es. "## 2. Quick Start", "## Extending Converters")
- **Content to add/modify:** il vero testo da inserire (NO "TODO: docs"), già finito
- Se il sub-task non ha impatto user-facing: `none in this task` esplicito.

L'agente aggiorna README **PRIMA** di committare. Il commit include sia il codice sia la doc update.

### 0.4 Parallelization Rules

Vedi **Dependency DAG** in §1. Regole:
- Sub-task **senza dipendenze in pending** possono essere dispatchati in parallelo dall'orchestratore.
- Mai due agenti che modificano gli stessi file in parallelo (orchestratore controlla `Files: Create/Modify` di ogni sub-task).
- Sub-task che modificano `KbSearchService.php`, `KbChatController.php`, `DocumentIngestor.php`, `Composer.tsx` sono **mutex** (locked exclusively).
- Test E2E (`frontend/e2e/`) e migrations sono parallel-safe (file diversi).

### 0.5 Branch + Commit Cadence

- Branch base: `feature/v3.0` (sotto-branch di `feature/v3-platform`).
- Ogni macro-task lavora su un sotto-branch dedicato:
  - Macro 1: `feature/v3.0-pipeline-foundation`
  - Macro 2: `feature/v3.0-filters-enterprise`
  - Macro 3: `feature/v3.0-grounding-tier1`
  - Macro 4: `feature/v3.0-consolidation` (parte solo dopo merge di Macro 1+2+3 in `feature/v3.0`)
- Ogni sub-task = un commit atomico. Convenzione: `feat(v3.0/T<id>): <title>` o `fix(v3.0/T<id>): ...` o `test(v3.0/T<id>): ...`.
- Ogni macro-task → PR verso `feature/v3.0` quando tutti i sub-task `[x]`.

### 0.6 Per-Task PR + Copilot Review Loop (NEW — user requirement)

**Ogni sub-task termina con questo loop esatto, NON solo `git commit` locale.**

```
1. Verification Gate green (tutti i test PHPUnit/Vitest/Playwright passano 100%)
   ↓
2. git push origin <task-branch>
   ↓
3. gh pr create --base <macro-branch> --title "feat(v3.0/T<id>): ..." --body "<task summary + checklist>" --reviewer copilot
   ↓
4. Wait 4-9 minutes for Copilot review
   (ScheduleWakeup if running in agent loop, or sleep+poll if standalone script)
   ↓
5. gh pr view <pr-number> --json reviews,comments,statusCheckRollup
   Extract Copilot inline comments + review summary
   ↓
6. For each Copilot comment:
   - Read context, classify: bug | nitpick | wrong | doc-drift
   - Ignore "wrong" (push back with reply explaining), fix the rest
   ↓
7. If fixes were applied:
   git add -A && git commit -m "fix(v3.0/T<id>): address Copilot review"
   git push
   GOTO step 4 (wait for Copilot re-review, 1 more cycle MAX, after that escalate to user)
   ↓
8. CI green + Copilot APPROVED (no outstanding "request changes")
   ↓
9. gh pr merge <pr-number> --squash --delete-branch
   ↓
10. git checkout <macro-branch> && git pull
    ↓
11. Mark sub-task [x], append LESSONS.md entry, move to next task
```

**Escalation triggers (orchestrator must STOP and ask user):**
- Copilot requests changes after 2 fix cycles → ask user to review
- CI red after 2 attempts to fix → ask user
- Test gate fails 3 times → ask user
- Copilot review takes > 15 min → ask user (rate limit / outage?)

**Branch hygiene:**
- One sub-task = one commit = one PR. Squash-merge keeps `feature/v3.0` linear.
- `--delete-branch` cleans up the per-task branch after merge.
- Macro-task branch (`feature/v3.0-pipeline-foundation`, ecc.) is the BASE of the per-task PRs.
- Final macro-task PR (`feature/v3.0-X` → `feature/v3.0`) is opened only AFTER all its sub-tasks are merged.

```php
<?php
namespace App\Services\Kb\Pipeline;

use App\Services\Kb\Contracts\ChunkerInterface;
use App\Services\Kb\Contracts\ConverterInterface;
use App\Services\Kb\Contracts\EnricherInterface;
use Illuminate\Contracts\Container\Container;

final class PipelineRegistry
{
    /** @var ConverterInterface[] */
    private array $converters = [];
    /** @var ChunkerInterface[] */
    private array $chunkers = [];
    /** @var EnricherInterface[] */
    private array $enrichers = [];

    public function __construct(Container $app, array $config)
    {
        foreach ($config['converters'] ?? [] as $cls) {
            $this->converters[] = $app->make($cls);
        }
        foreach ($config['chunkers'] ?? [] as $cls) {
            $this->chunkers[] = $app->make($cls);
        }
        foreach ($config['enrichers'] ?? [] as $cls) {
            $this->enrichers[] = $app->make($cls);
        }
    }

    public function resolveConverter(string $mimeType): ConverterInterface
    {
        foreach ($this->converters as $c) {
            if ($c->supports($mimeType)) return $c;
        }
        throw new \RuntimeException("No converter registered for MIME type: {$mimeType}");
    }

    public function resolveChunker(string $sourceType): ChunkerInterface
    {
        foreach ($this->chunkers as $c) {
            if ($c->supports($sourceType)) return $c;
        }
        throw new \RuntimeException("No chunker registered for source type: {$sourceType}");
    }

    /** @return ConverterInterface[] */
    public function allConverters(): array { return $this->converters; }
    /** @return ChunkerInterface[] */
    public function allChunkers(): array { return $this->chunkers; }
    /** @return EnricherInterface[] */
    public function allEnrichers(): array { return $this->enrichers; }
}
```

```php
<?php
namespace App\Services\Kb\Converters;

use App\Services\Kb\Contracts\ConverterInterface;
use App\Services\Kb\Pipeline\ConvertedDocument;
use App\Services\Kb\Pipeline\SourceDocument;
use Smalot\PdfParser\Parser;
use Symfony\Component\Process\Process;

final class PdfConverter implements ConverterInterface
{
    public function name(): string { return 'pdf-converter'; }

    public function supports(string $mimeType): bool
    {
        return $mimeType === 'application/pdf';
    }

    public function convert(SourceDocument $doc): ConvertedDocument
    {
        $start = hrtime(true);
        try {
            $parser = new Parser();
            $pdf = $parser->parseContent($doc->bytes);
            $pages = $pdf->getPages();
            $strategy = 'smalot';
        } catch (\Throwable $e) {
            $pages = $this->fallbackPdftotext($doc->bytes);
            $strategy = 'pdftotext';
        }

        $filename = basename($doc->sourcePath);
        $md = "# {$filename}\n\n";
        foreach ($pages as $i => $pageText) {
            $pageNum = $i + 1;
            $cleaned = $this->cleanText($pageText);
            $md .= "## Page {$pageNum}\n\n{$cleaned}\n\n";
        }

        $duration = (int) ((hrtime(true) - $start) / 1_000_000);
        return new ConvertedDocument(
            markdown: $md,
            mediaItems: [],
            extractionMeta: [
                'converter' => $this->name(),
                'duration_ms' => $duration,
                'page_count' => count($pages),
                'extraction_strategy' => $strategy,
                'source_path' => $doc->sourcePath,
            ],
            sourceMimeType: $doc->mimeType,
        );
    }

    private function fallbackPdftotext(string $bytes): array
    {
        $tmp = tempnam(sys_get_temp_dir(), 'pdf_');
        file_put_contents($tmp, $bytes);
        try {
            $process = new Process(['pdftotext', '-layout', '-enc', 'UTF-8', $tmp, '-']);
            $process->mustRun();
            $text = $process->getOutput();
            return preg_split("/\f/", $text) ?: [$text];
        } finally {
            @unlink($tmp);
        }
    }

    private function cleanText(string $text): string
    {
        // Normalize whitespace, strip null bytes
        $text = str_replace("\0", '', $text);
        $text = preg_replace('/[ \t]+/', ' ', $text);
        $text = preg_replace("/\n{3,}/", "\n\n", $text);
        return trim($text);
    }

    private function getPagesFromPagesObjects(array $pages): array
    {
        return array_map(fn($p) => $p->getText(), $pages);
    }
}
```

**Formula** (locked in design §2.2):
```
confidence = 100 * (
    0.40 * mean_top_k_similarity +
    0.20 * threshold_margin +
    0.20 * chunk_diversity +
    0.20 * citation_density
)
```
Where:
- `mean_top_k_similarity`: media degli `vector_score` dei chunk primary (0-1)
- `threshold_margin`: `(min_used_score - min_threshold) / (1 - min_threshold)` (0-1)
- `chunk_diversity`: `distinct_documents / total_chunks_used` (0-1)
- `citation_density`: `min(1, citations_count / max(1, words_in_answer / 100))` (0-1)
