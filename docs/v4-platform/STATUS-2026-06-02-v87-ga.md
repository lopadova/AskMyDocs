## New schema
`kb_synonyms` · `kb_doc_analyses` (both tenant-aware, R30/R31, in both architecture enumerations).

## New events / commands / endpoints
- Notification events: `kb_doc_stale_review`, `kb_doc_analysis_ready`.
- Commands: `kb:stale-review-sweep`, `notifications:digest-weekly`, `kb:prune-archived-versions` (+ 3
  scheduler slots, env-tunable).
- Admin API: `kb/synonyms` (CRUD), `kb/analyses` (read), `kb/documents/{id}/versions` + `/versions/diff`
  + `/restore-version`. Admin SPA: Synonyms, Doc Insights, Time Machine.

## Defaults / cost posture
- Synonym expansion: default ON, no-op without groups.
- Stale-review + weekly digest: scheduler slots default ON; per-user/per-channel via the existing
  preference matrix.
- AI deep-analysis: master `KB_CHANGE_ANALYSIS_ENABLED` ON; **canonical docs default ON, non-canonical
  opt-in** (`KB_CHANGE_ANALYSIS_NON_CANONICAL`); debounced on stable `(project, slug)`. Suggest-only.
- Time Machine retention: `KB_KEEP_ARCHIVED_VERSIONS` (default 10).

## Deferred (documented)

(config-level for now) · `MaintenanceCommandController` description coupling stays manual.
