# Feature Catalog: padosoft/laravel-pii-redactor v1.2.0 + admin v1.0.2

**Purpose:** Deterministic, EU-first PII redaction engine for Laravel 12/13 — 21 checksum-validated detectors, 4 strategies, 3 token stores, 3 NER drivers, 6 v1.2 admin-readiness APIs, separate React admin SPA.

> Generated by background Explore agent during v4.2 cycle (2026-05-09). Use this catalog as the integration spec for sub-PR 4.5 (boundary coverage extension) + sub-PR 5 (admin SPA).

## Executive Summary

- **21 checksum-validated detectors** across 4 always-on + 3 country packs (Italy, Germany, Spain; France/Netherlands/Portugal candidates)
- **4 pluggable strategies** (Mask, Hash, Tokenise, Drop) with persistent reverse-map storage
- **3 token store drivers** (InMemory, Database, Cache) for cross-process token recovery
- **3 NER drivers** (Stub, HuggingFace, spaCy) for optional ML-based supplemental detection
- **YAML custom-rule packs** for tenant-specific identifiers
- **6 new v1.2 admin-ready headless APIs** enabling safe runtime introspection
- **Standalone Laravel 13 + React + Vite admin SPA** with 8 API endpoints, 3 authorization gates, 7 screens, audit trail
- **Zero external services by default** — all detectors offline, deterministic, testable
- **GDPR + EU AI Act ready** — never exposes salt/keys/PII; counts-only audit events

## PII Detection Engine

**Location:** `vendor/padosoft/laravel-pii-redactor/src/RedactorEngine.php`

| Method | Purpose | Input | Output |
|---|---|---|---|
| `register(Detector $d)` | Register detector | Detector | `$this` |
| `detectors()` | List registered | none | `array<Detector>` |
| `scan(string $text)` | Detect without redaction | text | `DetectionReport` |
| `redact(string $text, ?Strategy)` | Detect + apply strategy | text + optional override | redacted string |
| `withStrategy(Strategy)` | Override default | strategy | `$this` |
| `withAuditTrail(bool)` | Toggle audit dispatch | bool | `$this` |

**Overlap resolution algorithm (deterministic):**
- Sort detections by offset ASC, length DESC
- Keep detector if earlier offset OR same offset but longer length OR previous detection ends before this offset
- Earlier offsets always win

**Audit dispatch:** When `audit_trail.enabled=true` AND `scan()` finds ≥ 1 detection, dispatches `PiiRedactionPerformed` event carrying `countsByDetector`, `total`, `strategyName`, `timestamps`. Never raw PII / output / salt.

## 4 Redaction Strategies

| Strategy | Output Format | Algorithm | Reversible | Priority |
|---|---|---|---|---|
| **MaskStrategy** | `[REDACTED]` (constant) | Constant string substitution | No | MUST (default) |
| **HashStrategy** | `[hash:abc123ef01234567]` | `SHA256(salt + detector + original)`; deterministic | Lookup-table only | SHOULD |
| **TokeniseStrategy** | `[tok:detector:abc123ef01234567]` | Deterministic token; persists in `TokenStore`; reversible via `detokenise()` | Yes (with store access) | SHOULD |
| **DropStrategy** | Empty string | Match removed entirely | No | COULD |

**Strategy switching:**
```php
Pii::redact($text, new HashStrategy($config));
Pii::withStrategy(new TokeniseStrategy(...))->redact($text);
```

**HashStrategy config knobs:**
- `hash_hex_length` (default 16, valid 8-64)
- `salt` (REQUIRED, env `PII_REDACTOR_SALT`)

**TokeniseStrategy:** stores token_id → original in `TokenStore`. `dumpMap()` / `loadMap()` for backup/restore across deploys.

## 3 Token Store Drivers

**Interface:** `TokenStore` exposes `put($token, $original)`, `get($token): ?string`, `has()`, `clear()`, `dump()`, `load()`. Used only by `TokeniseStrategy`.

| Driver | Backend | Persistence | Multi-Process | TTL | Priority |
|---|---|---|---|---|---|
| **InMemoryTokenStore** | PHP array | Process lifetime | No | No | MUST (default) |
| **DatabaseTokenStore** | `pii_token_maps` table (Eloquent `PiiTokenMap`) | Permanent until truncated | Yes | No (manual cleanup) | SHOULD |
| **CacheTokenStore** | Laravel cache (Redis/Memcached/DynamoDB/array) | TTL-based | Yes (shared cache) | Yes (`cache.ttl`) | SHOULD |

**DatabaseTokenStore schema:** `id`, `token` (unique), `original_value` (encrypted via cast if configured), `created_at`, `updated_at`.

**CacheTokenStore:** maintains `__pii_tokens:index` key for efficient `dump()` without keyspace scan.

## 3 NER Drivers (Optional ML supplementation)

**Interface:** `NerDriver::recognize($text): array<Detection>`. Fail-open (HTTP errors don't block deterministic redaction); merge results into same overlap-resolution pipeline.

| Driver | Backend | Cost Model | Priority |
|---|---|---|---|
| **StubNerDriver** | No-op (returns `[]`) | Free | MUST (default) |
| **HuggingFaceNerDriver** | HuggingFace Inference API; models like `dslim/bert-base-multilingual-cased-ner`, `xlm-roberta-large-finetuned-conll03-english` | Per-call (1000/month free, then per-request) | COULD |
| **SpaCyNerDriver** | Self-hosted spaCy HTTP server (`POST /` with JSON `{text}`); models `en_core_web_md`, `de_core_news_md`, `es_core_news_md`, `it_core_news_md` | Operational (one-time download + server infra) | COULD |

**Entity groups redacted:** PER (person), ORG, LOC, GPE, FAC

## Country Pack Architecture

**`PackContract` interface:**
```php
interface PackContract
{
    public function name(): string;          // 'italy', 'germany'
    public function countryCode(): string;   // ISO 3166-1 alpha-2
    public function description(): string;
    public function detectors(): array;      // list of Detector instances
}
```

**`DetectorPackRegistry`:** Boots all packs in `config('pii-redactor.packs')`, instantiates each, registers all detectors into `RedactorEngine`.

**Custom pack recipe:**
1. Implement detectors in `src/Packs/Iceland/Detectors/...`
2. Wrap in `PackContract` (`IcelandPack` with `countryCode() = 'IS'`)
3. Register in `config/pii-redactor.php` `packs` array

## YAML Custom Rule Packs

**`YamlCustomRuleLoader::load($filePath): array<CustomRuleSet>`**

```yaml
rules:
  - name: custom_albo_professional_id
    pattern: '/ISCR-[A-Z0-9]{8}/'
    flags: 'u'
  - name: custom_project_code
    pattern: '/[A-Z]{2}-\d{4}-[A-Z0-9]{6}/'
```

**Auto-registration:**
```php
'custom_rules' => [
    'auto_register' => true,
    'packs' => [
        storage_path('app/pii-rules/it-albo.yaml'),
        storage_path('app/pii-rules/project-codes.yaml'),
    ],
]
```

Validation: PCRE syntax check; invalid → `CustomRuleException` on first match attempt.

**AskMyDocs use:** Admin SPA "Overview" health card; integration diagnostics. Priority: MUST.

### 2. RedactionStrategyFactory — `src/Strategies/RedactionStrategyFactory.php`

Public builder; safe instantiation with validated config.

```php
$factory->names(): array  // ['mask', 'hash', 'tokenise', 'drop']
$factory->make(?string $name): RedactionStrategy
// Throws InvalidStrategyException if salt missing for hash/tokenise; or unknown name
```

**AskMyDocs use:** Admin SPA "Playground" strategy switcher. Priority: SHOULD.

### 3. DetectionReportFormatter — `src/Reports/DetectionReportFormatter.php`

Sample-masked report formatter for admin display.

```php
$formatter->safeArray(DetectionReport $report, bool $includeRawSamples = false): array
// Returns: ['total' => 5, 'isEmpty' => false, 'counts' => [...], 'samples' => [...masked or raw...]]
```

**Default:** all samples masked. Trust `$includeRawSamples=true` only when user has `view-raw-pii-samples` gate.

**AskMyDocs use:** Admin SPA Playground (safe-by-default redaction preview); audit reports. Priority: SHOULD.

### 4. TokenResolutionService — `src/TokenStore/TokenResolutionService.php`

Detokenise tokens to originals via configured `TokenStore`. Only fetches tokens referenced in text (no `dump()` call).

```php
$service->resolveToken('abc123ef01234567'): ?string  // 'john@example.com' or null
$service->detokeniseString('Email: [tok:email:abc123ef01234567]'): DetokeniseResult
```

**AskMyDocs use:** Admin SPA "Detokenise" screen for forensics; gate behind `detokenise-pii-tokens`. Priority: SHOULD.

### 5. CustomRulePackInspector — `src/CustomRules/CustomRulePackInspector.php`

Diagnostics for custom rule YAML packs (load health, rule count, parse errors). Doesn't modify the engine.

```php
$inspector->configuredPacks(): array
// Each entry: ['name', 'path', 'exists', 'readable', 'rule_count', 'valid', 'error']
```

**AskMyDocs use:** Admin SPA "Custom Rules" health screen. Priority: SHOULD.

### 6. DetokeniseResult — `src/Reports/DetokeniseResult.php`

Immutable DTO for detokenise operation. Methods: `output()`, `tokenCount()`, `resolvedCount()`, `unresolvedTokens()`, `toArray()`.

**AskMyDocs use:** Type-safe detokenise endpoint return value. Priority: SHOULD.

## Service Provider Hooks

`PiiRedactorServiceProvider::boot()` sequence:
1. Publish config, migrations, assets
2. Register `Pii` facade
3. Boot engine: register always-on detectors → boot country packs (`DetectorPackRegistry`) → auto-load custom rule packs
4. Register Artisan commands
5. `bootstrapHooks()` (subclass override hook for tenant-specific setup)

**Subclass extensibility example:**
```php
class CustomPiiRedactorServiceProvider extends PiiRedactorServiceProvider
{
    protected function bootstrapHooks(): void
    {
        if ($tenant = app('tenant')) {
            Pii::extend('custom_tenant_id', new CustomRuleDetector(
                'custom_tenant_id',
                (new YamlCustomRuleLoader())->load($tenant->pii_rules_path)
            ));
            
            // Per-tenant token store table
            app()->bind(TokenStore::class, fn() =>
                new DatabaseTokenStore(table: 'pii_token_maps_' . $tenant->id)
            );
        }
    }
}
```
