## 3. R37 GA execution

Per R37 (one merge per major release):

```bash
git checkout main
git merge --no-ff feature/v6.0 -m "release(v6.0.0): AI Act Compliance Bundle GA"
git tag v6.0.0 -m "v6.0.0 — AI Act Compliance Bundle (DSAR + Bias + Risks + Incidents + Consent + Disclosure + Cybersecurity + Attestation) + 8 pixel-ported admin SPA screens + host compliance depth"
```

The merge is `--no-ff` so the integration history (every Wn commit,
every rc tag) survives on main as a single grouped merge bubble. The
v6.0.0 tag pins the merge SHA.

`feature/v6.0` is preserved for any v6.0.x patches per R37.
