# Feature Catalog: padosoft/eval-harness-admin v1.0.0

**Package Name:** `padosoft/eval-harness-ui`
**Purpose:** Read-only React/TypeScript SPA admin panel consuming the `padosoft/eval-harness` Report API. 7 operational screens shipped as a Vite-powered SPA.

> Generated by background Explore agent during v4.2 cycle (2026-05-09). Use this catalog as the integration spec for sub-PR 7.

**Key Design Principles:**
- **Zero mutations** — read-only by design; no POST/PUT/PATCH/DELETE endpoints
- **API-driven** — consume only eval-harness Report API; no internal eval-harness code
- **SPA architecture** — React + TypeScript + Vite; Blade entrypoint + JSON bootstrap
- **Host-owned auth** — no bundled auth; delegated to host app via `can:eval-harness.viewer` gate
- **Tenant-aware** — optional `X-Eval-Harness-Tenant` header for multi-tenant deployments
