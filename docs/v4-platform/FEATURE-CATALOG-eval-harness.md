# Feature Catalog: padosoft/eval-harness v1.2.0

**Purpose:** Laravel-native evaluation framework for RAG/LLM regression testing. Golden datasets in YAML, nine built-in metrics, standalone output assertions, Markdown/JSON reports, and Artisan CI gates.

> Generated by background Explore agent during v4.2 cycle (2026-05-09). Use this catalog as the integration spec for sub-PRs 4 (eval-harness CI gate) and 7 (eval-harness-admin SPA).

## NINE BUILT-IN METRICS

| Feature | What it does | Where AskMyDocs benefits | Priority |
|---|---|---|---|
| **exact-match** | Case-sensitive string equality (1.0 if match, 0.0 else) | Rigid outputs: commands, email templates, system messages | SHOULD |
| **contains** | Substring match (1.0 if actual contains expected) | Partial matching: lists, bullet sets where order varies | COULD |
| **regex** | RegExp match (1.0 if pattern matches) | Format validation: URLs, emails, phone numbers, structured data | COULD |
| **rouge-l** | ROUGE-L recall (longest common subsequence) | Abstractive summaries; catch hallucinations via recall drop | COULD |
| **citation-groundedness** | Verify citation markers match evidence spans from metadata | Prevent unsupported citations; enforce [policy:refunds] → quote text match | MUST |
| **cosine-embedding** | Embed expected/actual; return cosine similarity | Semantic drift: "What time does store open?" vs "9am-5pm" — gap signals lost paraphrase tolerance | MUST |
| **bertscore-like** | Cross-encoder similarity via EmbeddingClient | Bidirectional alignment; superior to cosine for nuanced factuality | SHOULD |
| **llm-as-judge** | Call judge with temp=0, seed=42, JSON schema (strict parsing) | Subjective grading: helpfulness, tone, refusal quality; strict JSON rejects malformed | MUST |
| **refusal-quality** | Judge outputs refusal readiness (requires metadata.refusal_expected + metadata.refusal_policy) | Safety regression: jailbreak/prompt-injection samples; gate on pass_rate drop | MUST |

## BATCH PROFILES & EXECUTION

| Feature | What it does | Where AskMyDocs benefits | Priority |
|---|---|---|---|
| **--batch-profile=serial** (default) | SerialBatch: single process, sequential, no queue | Dev/feature branches: <100 samples, fast feedback, deterministic | SHOULD |
| **--batch-profile=smoke** | Light preset: serial, generous timeouts, relaxed rate limits | Developer inner loop: 10-20 samples, 30s, quick feedback | SHOULD |
| **--batch-profile=ci** | Aggressive: lazy-parallel, 4 workers, 30s timeout, checkpoints every 25 | PR gate: 30-200 samples, fail fast, checkpoint visibility | MUST |
| **--batch-profile=nightly** | Throttled: lazy-parallel, 16 workers, 60 samples/60s rate limit, 100-sample checkpoints | Nightly: 500+ samples, rate-limited, persistent manifests | SHOULD |
| **--batch=lazy-parallel** | Queue-backed parallel dispatch; producer chunks, workers process, cache collect | Spread load across Horizon; producer backpressure prevents queue explosion | MUST |
| **--chunk-size=N** | Producer dispatch window (≤ concurrency); tighter backpressure | Set chunk-size=2 for cost control: only 2 samples in flight despite concurrency=4 | COULD |
| **--rate-limit=N --rate-window-seconds=W** | Throttle to N samples per W-second window (monotonic clock, O(1) math) | Rate-limit to provider quota: 60 samples/60s for OpenAI embeddings | SHOULD |
| **--checkpoint-every=N** | Emit progress event every N samples; optional BatchProgressReporter binding | Integrate dashboards: Slack/webhook update every 25 samples | COULD |

## ARTISAN COMMANDS

| Feature | What it does | Where AskMyDocs benefits | Priority |
|---|---|---|---|
| **`php artisan eval-harness:run <dataset>`** | Core: load dataset, invoke SUT, score metrics, render report | PR gate: `eval-harness:run rag.factuality --registrar=... --json --out=eval.json` (exit 0 = success) | MUST |
| **`php artisan eval-harness:adversarial`** | Red-team: 10 categories (prompt-injection, jailbreak, pii-leak, ssrf, sql-injection, ascii-smuggling, competitor-endorsement, excessive-agency, hallucination, tool-abuse); save manifest | Nightly safety gate: 50 jailbreak + 50 prompt-injection + 50 hallucination; `--regression-gate` fails if refusal-quality.pass_rate drops >5% | MUST |
| **`--registrar=FQN`** | Class that registers datasets + binds SUT callable | AskMyDocs Registrar: loads 3 YAML datasets, wires KnowledgeAgent::answer() | MUST |
| **`--batch=serial\|lazy-parallel`** | Batch mode (overrides profile) | Local: serial; nightly: lazy-parallel --concurrency=4 --queue=evals | MUST |
| **`--batch-profile=ci\|smoke\|nightly\|custom`** | Named preset (applies timeout, rate-limit, checkpoint defaults) | Profile-based defaults; explicit flags always win | MUST |
| **`--concurrency=N`** | Worker fan-out cap (4 default) | Tune per env: prod=8, staging=4, CI=2 | SHOULD |
| **`--queue=name`** | Queue name for lazy-parallel (default) | Route evals to dedicated queue: --queue=evals-nightly | SHOULD |
| **`--timeout=seconds`** | Per-sample job timeout (30s default from profile) | Abort slow SUT; increase for complex reasoning | SHOULD |
| **`--batch-timeout=seconds`** | Producer wall-clock cap (180s default) | Fail fast if workers stall; increase for rate-limited dispatch | SHOULD |
| **`--rate-limit=N --rate-window-seconds=W`** | Throttle to N samples per W seconds | --rate-limit=60 --rate-window-seconds=60 for OpenAI embeddings | COULD |
| **`--json`, `--markdown`** | Report format (Markdown terminal, JSON to file) | --json for CI artifact; --markdown for readable output | SHOULD |
| **`--out=<path>`** | Write report to disk (reports.disk + reports.path_prefix; or --raw-path absolute) | Store eval-report.json; later consumed by dashboard | SHOULD |
| **`--outputs=<path>`** | Score pre-computed JSON/YAML outputs (bypasses SUT) | Reuse outputs from batch job; compare metrics without re-running | SHOULD |
| **`--category=prompt-injection --category=pii-leak`** (adversarial) | Red-team categories (repeatable) | Mix categories in one run | SHOULD |
| **`--metric=refusal-quality`** (adversarial) | Scoring metric for adversarial (default: refusal-quality) | Override: --metric=llm-as-judge for custom jailbreak grading | COULD |
| **`--manifest=<path>`** (adversarial) | Persist manifest JSON with run history; enables regression-gate | Store storage/app/adversarial-runs.json; manifest retains bounded history + clean baselines | SHOULD |
| **`--manifest-retain=N`** (adversarial) | Keep N newest manifests + clean baselines for compatible slices | --manifest-retain=10: keep 10 dated summaries; size based on distinct category/sample-count combos | COULD |
| **`--regression-gate`** (adversarial) | Compare current with latest failure-free baseline; fail if drop exceeds threshold | Gate: fail if refusal-quality.pass_rate regresses >5% from baseline | SHOULD |
| **`--regression-metric=metric:pass_rate`** (adversarial) | Which metric aggregate to gate on (repeatable) | --regression-metric=refusal-quality:pass_rate --regression-metric=llm-judge:mean | COULD |
| **`--promote-failures=<path>`** (adversarial) | Export low-scoring samples to YAML seed for future regression testing | Close gap: adversarial finds edge case → export to golden dataset → next PR run | COULD |

## YAML DATASET SCHEMA (eval-harness.dataset.v1)

| Feature | What it does | Where AskMyDocs benefits | Priority |
|---|---|---|---|
| **`schema_version: eval-harness.dataset.v1`** | Dataset contract versioning (defaults to v1 if omitted) | Future-proof datasets; allows additive schema evolution | COULD |
| **`name: rag.factuality.fy2026`** | Dataset identifier (used in reporting, trend grouping, --out default) | Datasets scoped by intent; git-diffable PRs show which samples changed | SHOULD |
| **`samples: [...]`** | Array of eval samples (30-200 typical) | Standard YAML list; sample count determines eval duration | MUST |
| **`samples[].id`** | Unique sample identifier within dataset | Used in CSV export, failure reporting; git-visible so PRs show delta | MUST |
| **`samples[].input: {...}`** | Callable input dict (SUT-dependent structure) | E.g., { "question": "What is the refund policy?" } | MUST |
| **`samples[].expected_output`** | Ground truth output (string or list) | Metrics compare actual_output against this | MUST |
| **`samples[].metadata.tags: [...]`** | Optional tag list for cohort grouping | Tag: ["easy", "support", "urgent"]; reports aggregate by tag | SHOULD |
| **`samples[].metadata.citation_evidence: [{citation, quote}, ...]`** | Optional evidence spans for citation-groundedness | Strict grounding: [policy:refunds] marker + exact quote text match in actual output | COULD |
| **`samples[].metadata.refusal_expected: bool`** (adversarial) | Whether sample should be refused (for refusal-quality metric) | Jailbreak: true; normal query: false; metric validates model chose appropriately | SHOULD |
| **`samples[].metadata.refusal_policy: string`** (adversarial) | Why refusal is expected (e.g., "user escalation", "pii-leak") | Makes grading rubric clear to judge; compliance reporting groups by policy | COULD |
| **`samples[].metadata.adversarial: {category, label, severity}`** (adversarial) | Adversarial metadata | Categorize: prompt-injection / jailbreak / hallucination; report aggregates by category | SHOULD |
